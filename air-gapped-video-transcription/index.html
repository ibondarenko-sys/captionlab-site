What this does and doesn't cover

This page describes CaptionLab's technical network behavior honestly, not a formal certification. CaptionLab is not independently certified or tested for classified/air-gapped government environments — if you work in one, verify against your own environment's specific security requirements before relying on it.

📡

No network calls during transcription

Once installed, generating subtitles doesn't attempt any outbound connection.

💽

Self-contained AI engine

The Whisper model, Python runtime, and ffmpeg all run from local files installed during setup.

🔁

Periodic license check only

The one thing that does reach out to the internet is an occasional license validity check — separate from transcription itself, and can go a couple weeks offline before it's required.

Setup still requires a network connection once

Installing CaptionLab's AI engine downloads roughly 3GB (Python runtime, ffmpeg, the Whisper model) from the internet the first time. That step needs connectivity. Everything after that — every transcription run — does not.

Frequently asked questions

Can I install CaptionLab on one machine and use the AI files on another with no internet at all?
CaptionLab's installer downloads its AI engine directly per-machine; it isn't currently packaged as a portable offline installer bundle for manual transfer. Contact editlabtools@gmail.com if this is a hard requirement for your setup.
Is CaptionLab certified for government or classified use?
No certification is claimed. It's a commercial Premiere Pro plugin; if you need a specific compliance certification, this likely isn't the right tool as-is.
Does the periodic license check block transcription if offline?
No — CaptionLab keeps working offline for a grace period between checks before requiring a reconnect, so a temporary loss of connectivity doesn't interrupt work.
